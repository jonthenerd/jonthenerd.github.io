IE8 Standards Mode and MOSS

October 26, 2009
Using IE8 Standards Mode with MOSS has been fairly trouble-free so far. However, the richhtmlfield editor throws javascript errors when the page is in IE8 Standards mode. Without switching to the Telerikcontrol or defaulting all of my pages to IE7 mode - there is a simpler fix!Enable IE7 compatability mode only on your edit pages. My master page had the IE8 meta tag:<meta equiv=“X-UA-Compatible” content=“IE=8”>right under the head element. To force your edit pages into IE7 mode, use the PublishingWebControls:EditModePanel to conditionally display it.
